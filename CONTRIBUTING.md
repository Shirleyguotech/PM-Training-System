# 贡献指南

感谢你愿意一起把“新人 PM 30 天训练系统”做成一个真正能帮到新人 PM 的开源学习工具。

这个项目欢迎两类贡献：

- 内容贡献：案例、自测题、模板、客户沟通话术、PM 判断规则、真实踩坑经验。
- 产品与代码贡献：页面体验、移动端适配、学习进度、数据导出、无障碍体验、文档优化。

## 贡献原则

请优先贡献能让新人 PM “看得懂、能练习、能复盘”的内容。

好的贡献通常具备这些特点：

- 场景具体：有客户、阶段、冲突、约束和 PM 动作。
- 贴近软件外包：范围、排期、变更、验收、客户反馈、售前承诺、技术风险。
- 可练习：新人能根据题目或案例做判断，而不是只看原则。
- 可复用：模板、话术、清单可以直接复制使用。
- 不泄密：不要包含真实客户名称、合同金额、账号、商业机密或敏感资料。

## 可以贡献什么

### 1. 新案例

适合补充这些类型：

- 售前承诺和交付范围不一致。
- 客户临时加需求。
- 开发评估与客户上线时间冲突。
- 验收阶段客户把新增需求当缺陷。
- 客户长期不反馈导致项目阻塞。
- 内部资源不足或质量问题影响交付。

建议格式：

```text
案例标题：
项目背景：
当前阶段：
当前状态：Green / Yellow / Red
主要干系人：
冲突点：
新人 PM 容易犯的错误：
建议 PM 动作：
可以产出的文档或话术：
```

### 2. 自测题

每道题建议包含：

```text
所属阶段：
题干：
A.
B.
C.
D.
正确答案：
解析：
```

题目要尽量像真实工作判断，不要只考概念定义。

### 3. 模板和话术

欢迎贡献：

- 售前交接确认表
- Kickoff 议程
- 需求澄清清单
- 变更评估模板
- 周报模板
- 验收推进邮件
- 客户不反馈提醒话术
- 项目复盘模板

模板应尽量可直接复制使用，并用占位符标明需要填写的内容。

### 4. 产品和代码

欢迎优化：

- 学习路径和阶段进度逻辑
- 自测题作答体验
- 案例库结构
- 移动端可读性和可点击性
- localStorage 数据导入导出
- README、贡献说明和发布文档

## 提交 Pull Request 前

请尽量确认：

- 页面可以正常打开。
- `script.js` 没有语法错误。
- 没有提交真实客户或公司敏感信息。
- 新增内容符合新人 PM 学习场景。
- 文案清楚、具体、不过度承诺。

如果你只想贡献一个想法，也可以先开 Issue，不需要直接写代码。

## 内容安全说明

请不要提交：

- 真实客户名称、项目名称、合同金额。
- 账号、密码、Token、内部链接。
- 未脱敏的聊天记录、会议纪要、合同或报价。
- 涉及个人隐私或公司机密的信息。

可以将真实经历改写为脱敏案例，例如：

```text
某 B2B 客户在 UAT 阶段提出新增审批流，希望纳入本期验收。
```

## 维护者说明

这个项目的初衷，是把 PM 新人在从 0 到 1 过程中遇到的困难，沉淀成可以练习、可以讨论、可以持续改进的学习工具。

如果你也曾经在项目交付里踩过坑、被客户追过、被范围变更困住过、在验收阶段反复拉扯过，欢迎把经验变成这个项目的一部分。
