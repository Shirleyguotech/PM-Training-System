# 如何参与贡献

感谢你愿意参与建设这个新人 PM 训练系统。

## 我需要什么样的帮助？

| 类型 | 具体内容 |
|------|----------|
| 📝 内容 | 补充案例库、自测题、场景话术 |
| 🧩 模板 | 增加新的 PM 模板（合同变更表、验收报告等） |
| 💻 代码 | 修复 bug、优化移动端、改善交互 |

## 第一次参与？从这里开始

1. 在 Issues 里找标签为 `good first issue` 的任务
2. 留言“我来认领”，我会分配给你
3. Fork 项目 → 修改 → 提交 Pull Request

## 内容贡献特别说明

如果你是 PM、交付经理或外包行业从业者，**完全不需要懂代码**也能参与：

- **提供真实案例**：工作中遇到的真实项目困境和解决方法（脱敏后即可），我会整理成网站的案例库，在 Issue 里直接粘贴即可，我会帮你整理成标准格式并在网站上标注你的署名。
- **出自测题**：针对某个阶段（售前交接、项目启动等）设计情景选择题，提供选项和解析。
- **写模板**：你工作中觉得好用的文档模板，直接分享出来。

## Pull Request 流程

1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/amazing-feature`)
3. 提交修改 (`git commit -m 'Add some feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 在 GitHub 上打开 Pull Request

## 疑问咨询

直接开 Issue 提问即可，我看到就会回复。

感谢每一份贡献 🙏
